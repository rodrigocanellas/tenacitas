\doxysubsection{Detailed Description}
\subsubsection*{template$<$cpt\+::event... t\+\_\+events$>$\newline
struct tenacitas\+::lib\+::alg\+::dispatcher$<$ t\+\_\+events $>$}

Class that allows creation of queues for an event, creation of subscribers for an event, and publising of events. 

The UML below, written in www.\+plantuml.\+com, shows the conceptual structure of the {\ttfamily dispatcher} and other associated structures

The central concept is an {\ttfamily event}, a struct that contains data about an interesting event in the application. It can be, for example, a incoming message from a network connection, a user menu choice or a temperature coming from a sensor.

A {\ttfamily \mbox{\hyperlink{namespacetenacitas_1_1lib_1_1typ_a5fff5d5a1f4666907090329156ecde3d}{tenacitas\+::lib\+::typ\+::subscriber}}} is a function object that will handle a {\ttfamily t\+\_\+event} object.

A {\ttfamily queue} is where {\ttfamily event} objects will be queued for subscibers objects to access them. For example, a user menu choice can generate a log message, a message sent to a remote program, and to change the color of widget in the user interface.

Besides, it is possible to define, for example, one single subscriber for the logging queue, five subscribers for the network message queue, and two for the widget color changing. When the user makes its menu choice, the five {\ttfamily \mbox{\hyperlink{namespacetenacitas_1_1lib_1_1typ_a5fff5d5a1f4666907090329156ecde3d}{tenacitas\+::lib\+::typ\+::subscriber}}} objects in the message network queue will \char`\"{}fight each other\char`\"{} to get the event to handle; the same for the two {\ttfamily \mbox{\hyperlink{namespacetenacitas_1_1lib_1_1typ_a5fff5d5a1f4666907090329156ecde3d}{tenacitas\+::lib\+::typ\+::subscriber}}} in the color changing queue.

It is possible to define a priority between the queues, so that an event is delivered to the highest priority queue before the others.

The {\ttfamily dispatcher} class is responsible for managing the creation of queue, adding {\ttfamily \mbox{\hyperlink{namespacetenacitas_1_1lib_1_1typ_a5fff5d5a1f4666907090329156ecde3d}{tenacitas\+::lib\+::typ\+::subscriber}}} to the queues, and publish event objects to the queues.

Please, look at the {\ttfamily Examples} section for examples on how to use these functions and classes.
