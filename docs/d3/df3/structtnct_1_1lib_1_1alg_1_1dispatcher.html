Detailed Description

template<tlc::event... t_events>
struct tnct::lib::alg::dispatcher< t_events >

Class that allows creation of queues for an event, creation of subscribers for an event, and publising of events.

The UML below, written in www.plantuml.com, shows the conceptual structure of the dispatcher and other associated structures

The central concept is an event, a struct that contains data about an interesting event in the application. It can be, for example, a incoming message from a network connection, a user menu choice or a temperature coming from a sensor.

A tenacitas::lib::subscriber is a function object that will handle a t_event object.

A queue is where event objects will be queued for subscibers objects to access them. For example, a user menu choice can generate a log message, a message sent to a remote program, and to change the color of widget in the user interface.

Besides, it is possible to define, for example, one single subscriber for the logging queue, five subscribers for the network message queue, and two for the widget color changing. When the user makes its menu choice, the five tenacitas::lib::subscriber objects in the message network queue will "fight each other" to get the event to handle; the same for the two tenacitas::lib::subscriber in the color changing queue.

It is possible to define a tld::event_priority between the queues, so that an event is delivered to the highest tld::event_priority queue before the others.

The dispatcher class is responsible for managing the creation of queue, adding tenacitas::lib::subscriber to the queues, and publish event objects to the queues.

Please, look at the Examples section for examples on how to use these functions and classes.
