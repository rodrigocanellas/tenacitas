breaker is the type of function that indicates if the loop should stop

Returns
status::ok if the loop where this function is being called should continue; concurrent::stop_loop if breaker wants the loop to stop in a expected way; or any other value, if the loop should stop for a unexpected reason

template<typename t_data >
typedef std::function<std::pair<status::result, t_data>)> tenacitas::concurrent::traits_t< t_data >::provider

provider is the type of function that provides data to the work function during the loop execution

Returns
status::ok, the parameter t_data has a meaningful data; status::stop_loop, then there is no more data to provide, and the parameter t_data has a default t_data value; anything else, it means there was an error in the provider function, and the parameter t_data has a default t_data value;
